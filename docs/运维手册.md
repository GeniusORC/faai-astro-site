## 1. 日常运维

### 1.1 内容更新（通知）
- 管理员通过 `/admin`（Decap）发布/编辑通知，写回 Git 后由托管平台自动重建。
- 如需紧急下线某条通知：在 Decap 删除并 Publish，或直接在 `src/content/notices/` 删除对应 `.md` 后推送。

### 1.2 页面文案修改
- 静态文案（如工委简介、需求库说明）位于 `src/pages/*.astro`，由维护者修改后重新构建/推送。

### 1.3 表单数据定期导出
- 定期访问 `https://<worker子域>/submissions?admin=<ADMIN_TOKEN>`，将表格导出存档（CSV/Excel）。
- 建议频率：每周或每次活动后，依提交量而定。

### 1.4 监控
- **可用性**：托管平台自带状态；可对正式域名做简单探活。
- **证书**：HTTPS 证书到期提醒（Let's Encrypt 通常自动续期，注意域名托管绑定）。
- **Worker 配额**：Cloudflare 免费档有请求/日限额，关注是否触顶。

## 2. 故障处理

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 前台页面打不开 | 托管构建失败 / 域名解析异常 | 查托管构建日志；确认 DNS 与 HTTPS 配置 |
| 构建失败 | 依赖未装 / Node 版本过低（<18.17）/ 通知 frontmatter 不符 schema | 本地 `npm run build` 复现；修正 frontmatter 或升级 Node |
| 表单提交无数据 | `formEndpoint` 未指向正式 Worker / Worker 未部署 / KV id 错 | 核对 `src/site.ts` 与 `worker/wrangler.toml`；`wrangler deploy` 重部署 |
| `/submissions` 403 | `ADMIN_TOKEN` 不正确 / 密钥未设 | 用 `wrangler secret put ADMIN_TOKEN` 重设 |
| Decap 登录失败 | `client_id`/`base_url` 错 / GitHub OAuth Callback URL 不匹配 | 核对 `public/admin/config.yml` 与 OAuth App 配置 |
| 发布通知前台不更新 | 托管未连 main / 构建未触发 | 确认仓库连接与分支；手动触发部署 |
| Worker 报错 | 密钥缺失（如 `GITHUB_CLIENT_SECRET`） | `wrangler secret put GITHUB_CLIENT_SECRET` |

## 3. 备份策略

| 对象 | 备份方式 | 说明 |
| --- | --- | --- |
| 站点代码与内容 | Git 仓库（GitHub） | 每次提交即版本备份，可回退任意历史 |
| 通知内容 | 同上（Markdown in `src/content/notices/`） | 天然纳入 Git |
| 表单提交数据 | KV 定期导出 + `/submissions` 表格存档 | 建议脚本化定时导出，避免单点丢失 |
| Worker 配置 | `worker/wrangler.toml` 纳入 Git（**不含明文密钥**） | 密钥用 `wrangler secret` 管理 |

## 4. 安全

- **密钥管理**：`ADMIN_TOKEN`、`GITHUB_CLIENT_SECRET` 仅以 `wrangler secret` 存储，禁止写入仓库或文档明文。
- **个人信息合规**：表单收集姓名/电话/邮箱，须告知用途并获授权；仅授权管理员可查看/导出；落实定期备份与最小知悉原则。
- **后台访问控制**：Decap 依赖 GitHub 账号体系，仅向可信委员授予仓库协作者权限。
- **提交防垃圾**：表单蜜罐 `_gotcha` 已启用；如垃圾增多，可加速率限制或验证码（需改 Worker）。
- **依赖安全**：定期 `npm audit` 与升级 Astro（见第 5 节）。

---

## 5. 升级与维护

- **Astro 升级**：`npm install astro@latest`，本地 `npm run build` 验证后推送；关注 Astro 5→6 的破坏性变更。
- **Decap 升级**：`public/admin/index.html` 中 `decap-cms` 版本号，按需提升并测试后台。
- **Worker 升级**：`wrangler` 与 `compatibility_date` 维护；改 `worker/index.js` 后 `wrangler deploy`。
