## 1. 引言

### 1.1 目的
明确企工委官网的功能性需求与非功能性需求，作为后续设计、开发、测试与验收的依据。

### 1.2 范围
覆盖官网前台（访客可访问的页面与表单）及内容管理后台（Decap CMS 发通知）。不含微信公众号、独立业务系统、付费 SaaS 后台。

### 1.3 术语
- **SSG**：静态站点生成（Static Site Generation），构建期生成纯 HTML。
- **Decap CMS**：开源、自托管的无头内容管理系统，通过写回 Git 仓库管理内容。
- **Worker / KV**：Cloudflare Workers 边缘函数与 KV 键值存储，用于承接表单提交。
- **企工委**：福建省人工智能学会企业工作委员会，简称「AI 企工委」。

### 1.4 参考文档
- `astro-site/SETUP.md`（Decap 接入与上线步骤）

- `astro-site/src/site.ts`、`astro.config.mjs`、`src/components/Form.astro`、`public/admin/config.yml`、`worker/index.js`、`worker/wrangler.toml`

## 2. 项目概述

### 2.1 背景
仿照中国人工智能学会官网（caai.cn）版式，建设「福建省人工智能学会·企业工作委员会」独立官网，提供需求对接、专家登记、入会申请、课题申报与活动交流一站式服务。

### 2.2 目标
- 开源建站、可自托管，无 SaaS 订阅费。
- 国内访问快、微信内置浏览器体验好。
- 五大业务模块完整可用，内容可由非技术人员维护。

### 2.3 用户角色
| 角色 | 说明 | 主要使用功能 |
| --- | --- | --- |
| 访客（企业/专家/拟入会者） | 未登录的网站浏览者 | 浏览首页与通知、提交专家登记/入会/课题表单 |
| 工委管理员 / 内容编辑 | 拥有 GitHub 与 Decap 权限的委员 | 通过 Decap 后台发布/编辑活动通知、导出表单数据 |
| 系统维护者（开发者） | 负责部署与运维 | 部署站点与 Worker、配置密钥、升级依赖 |

### 2.4 约束
- 部署于国内可访问节点，需 **ICP 备案 + HTTPS**（微信 JS-SDK 分享要求 HTTPS）。
- 表单含个人信息（姓名、电话、邮箱），需权限控制与备份，遵守个人信息保护要求。
- 技术栈以纯前端静态站为主，无自有数据库/服务端（表单后端由 Cloudflare Worker 承担）。

### 3.7 表单通用行为
- 原生 HTML `<form method="POST">`，无需 JavaScript，微信友好。
- 隐藏字段 `_form`（值=表单类型）、蜜罐 `_gotcha`（机器人填了静默丢弃）。
- 提交成功/失败跳转 `/thank-you?form=&status=`（status: ok / error / spam）。
- 备选：页面传 `surveyUrl` 时渲染「腾讯问卷」跳转按钮，不走 Worker。

### 3.8 内容管理（Decap CMS）
- 访问 `/admin`，GitHub OAuth 登录（经 Worker 代理）。
- 集合 `notices`：标题、发布日期、摘要、标签、封面图、正文（Markdown）。
- 发布即写回 `src/content/notices/*.md` 并触发托管平台重建。

## 4. 非功能需求

| 类别 | 需求 |
| --- | --- |
| 性能 | 纯静态页，首屏直出；目标首屏 < 1.5s（国内节点）；构建产物为 `dist/` |
| 兼容性 | 支持主流移动/桌面浏览器；重点适配微信内置浏览器（X5 内核） |
| 安全 | 表单蜜罐防垃圾；`/submissions` 需 `ADMIN_TOKEN`；个人信息仅管理员可导出 |
| 可维护性 | 内容（通知）与代码分离；非技术人员经 Decap 发稿；全栈仅 Astro + Worker |
| 可用性 | 移动端优先；导航与表单无错位；提交后有明确成功/失败反馈 |
| 合规 | ICP 备案 + HTTPS；个人信息收集需告知与授权、定期备份 |

---

## 5. 数据需求
- **通知内容**：Markdown 文件 + frontmatter，存于 `src/content/notices/`，纳入 Git 版本管理。
- **表单提交**：经 Worker 写入 Cloudflare KV（`FORM_SUBMISSIONS`），键为 `<formKey>-<时间戳>`，值为 JSON。
- **导出**：管理员访问 `/submissions?admin=<ADMIN_TOKEN>` 查看/导出。

---

## 6. 运行环境
- 构建：Node.js ≥ 18.17（推荐 20/22），`npm install` → `npm run build` 输出 `dist/`。
- 托管：任意静态托管（EdgeOne / Cloudflare Pages / Vercel / Netlify）。
- 表单后端：Cloudflare Worker + KV（独立部署，`wrangler deploy`）。

---

## 7. 验收标准
1. 首页与五大模块页面均可正常访问、移动端无错位。
2. 活动通知可经 Decap 后台发布并在前台更新。
3. 三张申请表单（专家/入会/课题）提交后跳转成功页，数据落入 KV。
4. 表单含 `ADMIN_TOKEN` 保护，未授权不可查看提交记录。
5. 站点以 HTTPS 提供，微信内打开正常。
6. 构建命令 `npm run build` 可成功产出 `dist/`。
