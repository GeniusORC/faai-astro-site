## 2. 测试策略
- **静态构建验证**：执行 `npm run build`，确认全部路由产出且工具链无误。
- **功能可达性**：逐页检查路由是否生成对应 HTML。
- **内容渲染**：通知集合 Markdown 是否正确生成详情页。
- **表单逻辑**：基于源码（`Form.astro` / `worker/index.js`）静态走查字段与数据流。
- **响应式/兼容**：依据断点与纯 CSS 导航设计进行走查（微信 X5 内核目标）。
- **安全**：蜜罐、提交记录鉴权、个人信息处理走查。

> 说明：表单端到端提交、Decap OAuth 登录、微信真机渲染需在**已上线环境**中由人工执行（见第 6 节未覆盖项）。

4. **ICP 备案 / HTTPS**：属上线合规动作，不在本构建测试中。
