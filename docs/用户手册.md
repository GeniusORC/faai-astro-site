## A 部分：终端用户（网站访客）

### A.1 浏览首页
- 打开官网首页，可见：**活动精选**轮播、**服务大厅**（五大模块入口）、**活动通知**最新列表。
- 顶部导航：首页 / 工委简介 / 企业需求库 / 专家登记 / 入会申请 / 课题申请 / 活动通知。
- 移动端：点击右上角「菜单」展开导航（纯 CSS，微信内无需额外加载）。

### A.2 查看活动通知
1. 点击导航「活动通知」，进入通知列表（按日期倒序）。
2. 点击任一标题，进入详情页阅读正文。
3. 首页「活动精选」与「活动通知」板块也提供最新内容入口。

### A.3 提交专家登记
1. 导航点击「专家登记」，填写：姓名*、职称/职务*、所在高校/科研机构*、研究方向*（下拉）、联系电话*、电子邮箱*、主要成果/简介（选填）。
2. 点击「提交申请」。
3. 提交成功跳转至结果页，提示「企工委工作人员将尽快与您联系」。

### A.4 提交入会申请
1. 导航点击「入会申请」，选择会员类型（企业会员/个人会员/团体会员）*，填写企业/个人名称*、联系人*、联系电话*、电子邮箱*、主营业务/研究方向（选填）。
2. 点击「提交申请」，等待结果页反馈。

### A.5 提交课题申请
1. 导航点击「课题申请」，填写课题名称*、申报单位/负责人*、课题类型*（联合攻关/企业委托/学生课题/开放课题）、联系电话*、电子邮箱*、课题摘要（选填）。
2. 点击「提交申请」，等待结果页反馈。

### A.6 企业需求库（当前说明）
- 进入「企业需求库」可查看需求类型说明（技术攻关/人才团队/场景数据合作）。
- **在线提交暂未开放**：现阶段请将需求发送至 `qigongwei@faai.org`，或关注「活动通知」中的线下需求对接会。（在线提交与检索为下一阶段功能。）

### A.7 提交结果说明
- 成功：`/thank-you?status=ok`，显示「提交成功」。
- 失败/垃圾拦截：`/thank-you?status=error` 或 `spam`，提示返回重试或联系企工委。
- 表单由本站免费自托管后端（Cloudflare Worker）接收，数据存于主办方自有账户，无第三方收费。

## B 部分：内容编辑 / 管理员

### B.1 访问后台
- 浏览器打开 `https://<正式域名>/admin/`（结尾斜杠不可少）。
- 首次使用需 GitHub 账号登录（OAuth 经 Worker 代理）。

### B.2 登录（GitHub OAuth）
1. 在 `/admin` 点击登录，跳转 GitHub 授权。
2. 授权范围 `repo`（用于写回内容仓库）。
3. 授权完成自动返回后台。

> 前置条件（由维护者配置）：GitHub 已建 OAuth App，其 Callback URL 为 `https://<域名>/admin/`；Decap 的 `client_id`、`base_url` 已指向正式值。

### B.3 发布活动通知
1. 左侧集合选择「活动通知」→ 点击 **New**（新建）。
2. 填写字段：
   - 标题（必填）
   - 发布日期（必填，datetime）
   - 摘要（可选）
   - 标签（可选，列表）
   - 封面图（可选，上传至 `public/assets/uploads` 并纳入 Git）
   - 正文（必填，Markdown）
3. 点击 **Publish**（发布）。
4. Decap 将 Markdown 写回 `src/content/notices/`，托管平台自动重新构建，**几分钟内前台更新**。

### B.4 编辑 / 删除通知
- 在「活动通知」列表选择某条 → 修改后 **Save**；或删除后重新发布。

### B.5 导出表单数据
- 管理员访问 `https://<worker子域>/submissions?admin=<ADMIN_TOKEN>`。
- 页面以表格展示提交记录（时间 / 类型 / 内容），可据此人工导出 CSV 或复制存档。
- 表单类型：`expert`（专家登记）、`member`（入会申请）、`project`（课题申请）。

### B.6 权限与备份提醒
- 表单含个人信息（姓名、电话、邮箱），仅授权管理员可查看与导出。
- 内容已纳入 Git 版本管理（天然备份）；表单数据建议**定期导出备份**（KV 导出或表格存档）。
- 遵守个人信息保护要求：不向无关方泄露提交者信息；密钥 `ADMIN_TOKEN`、`GITHUB_CLIENT_SECRET` 不得明文入库。

### B.7 本地预览后台（可选）
```bash
npx decap-server        # 终端 A：本地 Git 后端
npm run dev             # 终端 B：站点，开 http://localhost:4321/admin/
```
并在 `public/admin/config.yml` 取消 `local_backend: true` 注释（无需 OAuth 即可本地登录）。

## C 部分：常见问题（用户侧）

| 问题 | 处理 |
| --- | --- |
| 提交表单后一直转圈 | 检查网络；确认 `formEndpoint` 指向的 Worker 可用；重试或发邮件至 qigongwei@faai.org |
| 微信内页面错位 | 刷新；如仍错位请截图反馈给维护者（已做移动端优先适配） |
| 后台登录不进去 | 确认使用 GitHub 账号且为仓库协作者；Callback URL 与 `client_id` 是否正确 |
| 发布通知前台没变 | 等待几分钟；确认构建成功（可让维护者查托管构建日志） |
