## 1. 环境准备

| 依赖 | 版本要求 | 用途 |
| --- | --- | --- |
| Node.js | ≥ 18.17（推荐 20 / 22） | 构建 Astro 站点 |
| npm | 随 Node 自带 | 安装依赖 |
| wrangler | `npm i -g wrangler`（或项目 devDependency ^4.112.0） | 部署 Cloudflare Worker |
| Git | 任意 | 代码与内容版本管理 |
| Cloudflare 账号 | 免费档即可 | Worker + KV |
| 已备案域名（可选） | — | 绑定正式 HTTPS 域名，提升微信体验 |

> 代码仓库：`https://github.com/GeniusORC/faai-astro-site.git`（Public）。

## 2. 本地构建与预览

```bash
cd astro-site
npm install            # 安装依赖（首次或依赖变更时）
npm run dev            # 开发预览 http://localhost:4321
npm run build          # 构建到 dist/
npm run preview        # 本地预览 dist/ 产物
```

构建成功后，`dist/` 即为可部署的静态产物。

---

## 3. 托管部署（任选其一）

> 通用要点：构建命令 `npm run build`，输出目录 `dist/`，Node 版本 20。连接 Git 仓库后，push `main` 自动触发构建与上线。

### 3.1 腾讯云 EdgeOne / 静态托管
1. 新建静态站点/EdgeOne 项目，绑定 Git 仓库或上传 `dist/`。
2. 绑定已备案域名并开启 HTTPS。
3. 国内访问快，微信分享体验好。

### 3.2 Cloudflare Pages
1. 登录 https://pages.cloudflare.com → **Connect to Git** → 选 `faai-astro-site`。
2. Build command：`npm run build`；Build output directory：`dist`；Node.js version：20。
3. 部署完成后获得 `*.pages.dev` 域名。

### 3.3 Vercel / Netlify
- 同上：Framework 选 Astro（或自定义），Build `npm run build`，Output `dist`。

### 3.4 子路径部署（如 GitHub Pages 项目页）
- 在 `astro.config.mjs` 设置 `base: '/repo'`，其余不变。

## 4. Worker 部署（表单后端 + OAuth 代理）

表单数据经 Cloudflare Worker 写入 KV，Decap 的 GitHub OAuth 也由该 Worker 代理。

```bash
# 1. 登录 wrangler
wrangler login

# 2. 创建 KV 命名空间，记录返回的 id
wrangler kv namespace create FORM_SUBMISSIONS

# 3. 编辑 worker/wrangler.toml
#    - 将上一步 id 填入 [[kv_namespaces]] 的 id
#    - SITE_URL 改为正式站点域名（如 https://<project>.pages.dev）
#    - ADMIN_TOKEN 建议用密钥：wrangler secret put ADMIN_TOKEN
wrangler secret put ADMIN_TOKEN        # 输入强口令
wrangler secret put GITHUB_CLIENT_SECRET   # 输入 GitHub OAuth App 的 Client Secret

# 4. 部署
cd worker
wrangler deploy
```

部署后 Worker 子域形如 `https://faai-forms.<subdomain>.workers.dev`，提供以下路由：
- `GET /auth`、`GET /callback`：Decap GitHub OAuth 代理
- `POST /submit`：接收表单提交
- `GET /submissions?admin=<ADMIN_TOKEN>`：查看/导出提交记录（需口令）

## 5. 上线前配置清单

| # | 文件 | 配置项 | 改为 |
| --- | --- | --- | --- |
| 1 | `src/site.ts` | `formEndpoint` | 正式 Worker 子域 `https://<worker>/submit` |
| 2 | `astro.config.mjs` | `site` | 正式域名（OG/sitemap/绝对链接） |
| 3 | `public/admin/config.yml` | `client_id` | 自建 GitHub OAuth App 的 Client ID |
| 4 | `public/admin/config.yml` | `base_url` | 正式 Worker 子域（OAuth 代理） |
| 5 | `worker/wrangler.toml` | `SITE_URL` | 正式站点域名 |
| 6 | `worker/wrangler.toml` | `ADMIN_TOKEN` | 强口令（密钥） |
| 7 | `worker/wrangler.toml` | KV `id` | 正式 KV 命名空间 id |
| 8 | GitHub OAuth App | Callback URL | `https://<正式域名>/admin/`（结尾斜杠） |

> 表单备选：若不想部署 Worker，可在各表单页传 `surveyUrl`（腾讯问卷链接），页面自动显示「点击前往填写」按钮，数据存于腾讯问卷后台。

## 6. ICP 备案与 HTTPS

- 绑定国内可访问域名须完成 **ICP 备案**（服务商代提交）。
- 必须为站点启用 **HTTPS**（微信 JS-SDK 自定义分享卡片要求 HTTPS；至少保证域名 HTTPS 可访问）。
- 微信内打开体验以 HTTPS 域名最佳；JS-SDK 自定义分享为可选项。

---

## 7. 回滚

- **前台回滚**：静态托管平台均支持历史部署一键回滚（Cloudflare Pages / Vercel / Netlify 在控制台选择上一版本重新部署）。
- **内容回滚**：通知内容在 Git 中，回退对应 commit 并重新构建即可。
- **Worker 回滚**：`wrangler rollback`（或控制台选择历史版本）。

---

## 8. 验证部署

1. 访问正式域名首页，确认页面与导航正常。
2. 提交一张申请表单，确认跳转 `/thank-you?status=ok`。
3. 访问 `/submissions?admin=<ADMIN_TOKEN>`，确认数据可见。
4. 打开 `/admin`，确认可用 GitHub 登录并发布一条通知，几分钟后前台更新。
5. 用微信内置浏览器打开，确认无错位、HTTPS 正常。

## 9. 常见问题

| 现象 | 排查 |
| --- | --- |
| 表单提交后无数据 | 检查 `formEndpoint` 是否为正式 Worker 子域；Worker 是否部署成功；KV id 是否正确 |
| `/admin` 登录失败 | 检查 Decap `client_id`、`base_url`；GitHub OAuth App 的 Callback URL 是否为 `https://<域名>/admin/` |
| 发布通知前台不更新 | 确认托管已连接 Git main 分支且构建成功；检查通知 Markdown frontmatter 是否符合 schema |
| 构建失败 | 查看托管构建日志；常见为依赖未装或 Node 版本过低（需 ≥18.17） |
